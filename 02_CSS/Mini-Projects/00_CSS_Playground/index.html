Selectors

Types of Selectors

  • Universal Selector : * (Asterisk)
  • Element Selector : All tags in HTML (eg: h1,h6,p,span,div etc.)
    • We use comma (,) sign to apply same style in multiple selectors (eg: p,h1{....} )
  • ID Selector : eg: #id_name
  • Class selector: eg: .class_name
  • Attribute Selector : tag[attr="value"] eg: input[type="password"]{.....} or input[type="radio"]{....}
    here we can select elements based on the presence or value of a given Attribute

    • Practice Question

    Set the color for paragraph inside divs with the id "myId" to green

    Solve:

                #myId p{color: green;}
  • Descendent Selector : We use whitespace sign eg: div p{...} (selected every p inside every div )

  • Combinator: Combining multiple selectors

    types of Combinator

    • Adjecent sibling Combinator : We use + sign eg: p + h3 (here p and h3 are in the equal level or we can say they are siblings)
    • Child Combinator : we use > sign eg: p > h3 (here h3 is just under por we can say h3 is the child of p)

Spacificity calculation

Element < class < id
elements(included Pseudo-elements) = 1
class (included Pseudo-classes)= 10
id = 100
For example: 
            div .greetDiv .greet{ color : green}; 0 2 1
            .greetDiv p .greet{ color: blue}; 0 2 1
            .greetDiv .greet{ color: red}; 0 2 0
            higher spacificity = higher priority.... and if spacificity same we prioritize the last one
So the color will be blue

NB: Inline style is more specific then id

So, we can say [ inline > id > class > elements ]

Inheritance

Every elements try to inherit parent's CSS property. If we give any style to an element that style will be inherited by their child, grand-child and further generation.

Exception: input & button

So, in CSS we have to do the Inheritance manually, [ input button {background-color: inherit}; ]

Flexbox

its is a one dimensional layout method for arranging items in rows or columns in 1D form

for flex container

  • display: flex
  • flex-direction:
    • row
    • row-reverse
    • column
    • column-reverse

  • justify-content:
    • flex-start
    • flex-end
    • center
    • space-between
    • space-around
    • space-evenly

  • flex-wrap:

    sets whether flex items are forced onto one line or can wrap onto multipe lines

    • nowrap
    • wrap
    • wrap-reverse

  • align-items:

    Distribute our items along the cross size

    • flex-start
    • flex-end
    • center
    • baseline - text on the flex items will be on the same line

  • align-content:

    it sets the distribution of space between and around content items along a flexbox's cross axis

    • flex-start
    • flex-end
    • center
    • space-around
    • space-between
    • space-evenly
    • baseline- text on the flex items will be on the same line

for flex items:

  • align-self:

    align an item along the cross axis

    • flex-start
    • flex-end
    • center
    • baseline- text on the flex items will be on the same line
    align-self has always higher priority then align items so, [ align-self > align-items ]

Flex Sizing

  • flex-basis: px - it sets the initial main size of a flex item
  • flex-grow: ( default 1) - it specifies how much of the flex container's remaining space should be assigned to the flex item's main size
  • flex-shrink: ( default 1) - it sets the flex shrink factor of a flex item

Flex sizing Shorthand

  • flex-grow | flex-shrink| flex-basis

    • example: [ flex: 2 2 100px; ]

  • flex-grow | flex-basis

    • example: [ flex: 2 100px; ]

  • flex-grow (unitless)

    • example: [ flex: 2; ]

  • flex-basis

    • example: [ flex: 100px; ]
